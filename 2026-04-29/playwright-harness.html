Test infra

Playwright UI test harness landed

Until now, Advisor had unit tests for individual functions but nothing that launched the app and clicked through. That left a whole class of bugs — layout, keyboard shortcuts, tab transitions — outside the test net. Fixed overnight.

What shipped

  • Playwright with Electron support added as a dev dependency.
  • Harness at tests/ui/, runnable via npm run test:ui.
  • Tests launch the pre-built app bundle — the same one the installer ships — so they exercise real production code, not a dev-mode facsimile.
  • Bedrock mock: setting one environment variable short-circuits every AI-service call to deterministic stubs. Tests need no AWS credentials and make no live network calls.

Initial coverage

  • Smoke test per tab — does each one load?
  • Ambient strip renders with the expected content
  • The hidden-sidebar Code-tab layout is visually correct (regression protection for yesterday’s fix)
  • Cmd+1 through Cmd+5 keyboard shortcuts switch tabs correctly

Failure screenshots are annotated

When a test fails, the harness captures the screenshot and overlays arrows, boxes, and labels marking the broken region using sharp (an image-manipulation library). The Playwright report shows the annotated image, not just raw stdout. This is the part that makes UI test failures actionable — you see what broke, not just that it broke.

Two gotchas documented

  • The Bedrock-mock env var is read once at module load; flipping it mid-process has no effect.
  • Launching the tests from a non-GUI shell (certain sudo contexts) causes an Electron parent-death crash before the first window opens. Workaround is a small launchd wrap, documented in the implementation notes.

What this unlocks

Future backlog items add coverage as features ship. The harness is the floor, not a one-time deliverable. The “did this break anything visible?” question now has an automated answer instead of manual clicks.
