Item 3

Clickable Slack-to-Advisor links

A proper deep-link protocol between the Slack agent and the Advisor desktop app. Previously one hardcoded URL shape for one view type; now six view types with a documented, versioned protocol.

What you can click

In a Slack message from the bot, any of these six URL shapes will open the right view in Advisor:

  • A specific Claude Code session (opens the Code tab, activates that session)
  • A Slack-authored session (same as above, but self-documents it came from the bot)
  • A specific plan file (opens the Plans tab to that file)
  • A specific task (opens the Tasks tab to that task)
  • A specific day of Advisor thread history (opens Advisor to that day)
  • A project’s task list (opens Tasks, scrolls to that project’s group)

How it’s split

Two codebases, one contract. The Advisor repo holds the receiver (parses the URL, routes to the right tab). The Slack bot repo holds the sender (a helper function every message builder can use). The same url-schema.md document lives byte-identical in both repos — single source of truth.

If something goes wrong

Unknown URLs surface a clear toast in Advisor rather than crashing. Missing items (a task that’s since been removed, a plan whose path changed) also toast. Nothing silent.

Coordination

Planning went through a formal plan-mode flow with four decision points confirmed before implementation. Implementation landed in 9 commits across the two repos, each commit independently buildable and green.

What’s next

End-to-end click-through deferred until the next production rebuild of Advisor — dev-mode launch has a known environment issue on the current host. Unit tests + a URL-routing trace script cover the parser side fully.
