Item 7

Playwright UI test harness

A proper UI-level test harness for Advisor. Until now, we had unit tests for individual functions but nothing that actually launched the app and clicked things. That left a whole class of bugs — layout, keyboard shortcuts, tab transitions — outside the test net.

What landed

  • Playwright with Electron support, added as a dev dependency.
  • Harness at tests/ui/, runnable via npm run test:ui.
  • Tests launch the pre-built app bundle (the same one the installer ships) rather than a dev-mode facsimile, so they exercise real production code.
  • A Bedrock mock — setting one environment variable short-circuits every AI-service call to deterministic stubs. Tests require no AWS credentials, no live network.

Initial coverage

Smoke test per tab (does it load?), ambient strip renders with the expected content, Code tab’s hidden-sidebar layout is visually correct (regression protection for the fix shipped earlier today), and the Cmd+1 through Cmd+5 keyboard shortcuts switch tabs correctly.

Failure screenshots are annotated

When a test fails, the harness captures the screenshot and overlays arrows, boxes, and labels marking the broken region using sharp (an image-manipulation library). The Playwright report shows the annotated image, not raw stdout. This is the part that makes UI test failures actionable — you see what broke, not just that it broke.

Two gotchas documented

  • The Bedrock-mock env var is read once at module load; flipping it mid-process doesn’t help.
  • Launching the tests from a non-GUI shell (certain sudo contexts) causes an Electron parent-death crash before the first window opens. Workaround is a small launchd wrap, documented in the implementation notes.

Future coverage

New backlog items add coverage as features ship — the harness is the floor, not a one-time deliverable.
