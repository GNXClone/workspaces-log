Item 1

Reusable prompt library

Set up a dedicated folder in the Advisor repo for prompt macros — reusable instructions we hand to AI sessions for common, repeatable jobs. Before today these lived in chat scrollback; easy to lose, hard to share.

Where it lives

docs/prompts/ in the Advisor repo, on branch feat/pittsbot-deep-link-scheme. One .md file per prompt, plus a README.md index.

What’s in it today

  • build-mission-brief — the long-form template for dispatching a multi-hour autonomous build. Defines scope, done criteria, failure rules, report format.
  • smoke-test — post-build verification. Start the dev build, hit each tab, check log output, report findings.
  • write-user-docs — generate user-facing documentation when a capability ships or shifts.

Why it matters

Consistency. Every time a mission brief is written from scratch, it drifts. A stored version means the next build uses the same failure rules, the same report format, the same stop conditions — which compounds over time.
