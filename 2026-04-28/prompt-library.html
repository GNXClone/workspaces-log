What’s in it

  • build-mission-brief — long-form template for dispatching a multi-hour autonomous build. Defines scope, done criteria, failure rules, report format.
  • smoke-test — post-build verification. Start dev, hit each tab, check log output, report.
  • write-user-docs — generate user-facing documentation when a capability ships.
  • Placeholder for write-retrospective after the first meaningful milestone.

Why

  • Mission briefs used to be rewritten from scratch each time — they drift.
  • A stored version means the next build uses the same failure rules, the same report format, the same stop conditions. Compounds over time.
